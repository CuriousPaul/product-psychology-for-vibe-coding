# 윤리적 디자인 체크리스트

> "Ethics is a long-term strategy" — 윤리는 장기 전략이다.

---

## Humane Design Principles

| 원칙 | DO | DON'T |
|------|-----|-------|
| **Save Time** | 사용자의 시간을 절약해준다 | 시간을 낭비하게 만든다 |
| **Value Attention** | 주의를 존중한다 | 불필요하게 방해한다 |
| **Reflect Humane Values** | 인간적 가치를 반영한다 | 주주 이익만 반영한다 |

---

## 3가지 윤리 테스트

### 1. Regret Test (후회 테스트)
> "사용자가 지금 이 화면을 내 옆에서 보고 있다면..."

- 이 다크 패턴을 사용자가 알면 후회할까?
- 이 넛지가 사용자의 진짜 이익에 부합하는가?
- 사용자가 이 설계 의도를 알아도 괜찮은가?

### 2. Black Mirror Test (블랙 미러 테스트)
> "이 기능이 극단적으로 악용된다면..."

- 특정 지역/직업/가족형태에 불리한 조건이 되지 않는가?
- 의도하지 않은 구조적 차별이 발생할 수 있는가?
- AI/알고리즘이 편향을 증폭시킬 수 있는가?
- Second-order Effect (의도치 않은 부작용)를 고려했는가?

### 3. In Real-Life Test (실생활 테스트)
> "이 제품이 실제 사람이라면..."

- 친절한 도우미인가, 귀찮은 세일즈맨인가?
- 진심으로 돕는 친구인가, 이용하려는 사람인가?
- 사용자의 "No"를 존중하는가?

## 바이브코딩 시 윤리 체크 타이밍

| 시점 | 체크 항목 |
|------|-----------|
| **설계 시작** | Humane Design Principles 확인 |
| **넛지 적용 시** | Regret Test — 사용자가 알아도 괜찮은 넛지인가? |
| **개인화 적용 시** | Black Mirror Test — 차별적 결과가 나올 수 있는가? |
| **출시 전** | In Real-Life Test — 이 제품은 어떤 사람인가? |
| **A/B 테스트 설계 시** | Second-order Effect — 의도치 않은 부작용은? |

## 경계선 판단 기준

**허용되는 넛지:**
- 사용자의 실제 목표 달성을 돕는 설계
- 투명하게 공개해도 문제없는 설계
- 사용자가 쉽게 opt-out 할 수 있는 설계

**경계해야 할 다크 패턴:**
- 사용자의 실수를 유도하는 설계
- 숨겨진 비용이나 조건
- 취소/해지를 의도적으로 어렵게 만드는 설계
- 감정적 취약 상태를 악용하는 설계
- 거짓 희소성/긴급성 연출
