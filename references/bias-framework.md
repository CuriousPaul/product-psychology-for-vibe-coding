# B.I.A.S Framework 상세 가이드

> 논리로 만들고(System 2), 사용자는 본능으로 사용한다(System 1). 이 Gap을 메운다.

---

## B — Block: 뇌의 필터를 통과하라

뇌는 인지적 효율을 위해 3가지 정보를 본능적으로 차단한다:

### 차단되는 정보 3가지

| 유형 | 설명 | 관련 이론 |
|------|------|-----------|
| **High Effort** | 복잡하고 노력이 필요한 정보 | 힉의 법칙 (Hick's Law): 선택지가 많을수록 의사결정 시간 증가 |
| **Unrelated** | 나와 관련 없어 보이는 정보 | 선택적 주의 (Selective Attention): 목적과 연결된 정보에만 주의 |
| **Redundant** | 반복적이고 예측 가능한 정보 | 배너 무시 (Banner Blindness): 광고처럼 보이는 영역을 무시 |

### 주의를 끄는 3가지 요소

| 요소 | 설명 | 설계 전략 |
|------|------|-----------|
| **단기 기억 정보** | 방금 본 정보를 기준값으로 삼음 | 프라이밍 효과: 메인 CTA 전에 관련 정보를 미리 노출 |
| **신념/관심 관련 정보** | 나의 관심사와 일치하면 더 잘 보임 | 개인화: 이름, 취향, 이전 행동 기반 맞춤 |
| **예상치 못한 것** | 패턴을 깨는 의외의 요소 | 패턴 브레이크: 기존 반복 패턴을 깨는 인터랙션 |

### Block 체크리스트

- [ ] "이게 무슨 뜻이지?" 하고 1초 이상 고민하게 만드는 요소 없는가?
- [ ] 의식적 노력(시스템 2) 없이 직관적으로 이해 가능한가?
- [ ] 한 화면 내 선택지 6개 이하인가?
- [ ] '광고'처럼 보이지 않는가?
- [ ] 습관적으로 무시하는 영역(상단 배너, 우측 사이드바)에만 있지 않은가?
- [ ] 중요 정보가 실제 광고 옆에 배치되어 함께 무시당하지 않는가?
- [ ] 메인 메시지 전에 단기 기억에 관련 정보를 심어주었는가?
- [ ] "나와 관련된 정보"라는 확신을 주는 장치가 있는가?
- [ ] 기존 패턴을 깨는 새로운 요소가 있는가?

## I — Interpret: 가치를 명확하게 해석시켜라

### 7가지 Interpret 전략

| # | 전략 | 핵심 원리 | 코딩 적용 |
|---|------|-----------|-----------|
| 1 | **Familiarity** | 익숙한 패턴에 자동 반응 | 표준 UI 패턴 사용, 플랫폼 컨벤션 준수 |
| 2 | **Cognitive Load** | 인지 노력 초과 시 이탈 | 짧은 문단, 명확한 시각 요소, 최소 인지 노력 |
| 3 | **Clear Benefit** | 혜택 먼저 보여야 의미있게 해석 | 기능 설명 전에 "당신이 얻는 것" 먼저 |
| 4 | **Anchoring** | 먼저 제시된 정보가 기준점 | 가격표에서 비싼 플랜을 먼저, 할인 전 가격 표시 |
| 5 | **Loss Aversion** | 이득의 기쁨보다 손실의 고통이 2배 | "놓칠 수 있어요" 시각화, 카운트다운 |
| 6 | **Discoverability** | 핵심 기능을 쉽게 찾게 | 검색 필드 하단에 인기 카테고리 노출 |
| 7 | **Labor Illusion** | '당신을 위해 일하는 중' 인상 | 로딩 중 작업 과정 시각화 |

### Interpret 체크리스트

- [ ] 인지적 부담이 최소화되었는가?
- [ ] 익숙한 패턴을 기반으로 디자인되었는가?
- [ ] 비교 가능한 기준점을 제시했는가?
- [ ] 기다리는 시간을 가치있는 경험으로 바꾸었는가?
- [ ] 혜택을 명확히 전달하고 있는가?
- [ ] 핵심 기능이 쉽게 발견되는가?
- [ ] 손실을 시각적으로 보여주고 있는가?

## A — Act: 행동의 마찰을 제거하라

> 안다는 것과 한다는 것은 다르다. 가치를 이해해도 행동으로 이어지지 않을 수 있다.

### 행동하지 않는 2가지 이유

1. **높은 인지 부하** — "너무 복잡하고 어려워요"
2. **부족한 동기 부여** — "왜 지금 이걸 해야 하죠?"

### 전략 1: 마찰 줄이기 (Reduce Friction)

| 기법 | 설명 | 실전 적용 |
|------|------|-----------|
| **Remove Options** | 선택지 줄이기 | 당장 필요한 선택지만 남기고 세부는 다음 단계에서 |
| **Valid Defaults** | 유효한 기본값 | 최대 할인 쿠폰 자동 적용, 권장 옵션 미리 선택 |
| **Split Steps** | 단계 쪼개기 | 복잡한 과정을 작은 "Yes"로 분리. 쉬운 선택 먼저 |
| **Reveal Features** | 점진적 공개 | 처음엔 핵심만, 필요 시점에 추가 기능 노출 |

### 전략 2: 넛지 (Nudge)

| 기법 | 설명 | 실전 적용 |
|------|------|-----------|
| **Social Proof** | 사회적 증거 | "오늘 N명이 예약했습니다", 후기, 사용자 수 |
| **Curiosity Gap** | 호기심 자극 | 정보를 일부만 보여주고 "더 알아보기"로 유도 |
| **Scarcity** | 희소성 | "오늘만", "종료 임박", 남은 수량 표시 |

### Act 체크리스트

**마찰 점검:**
- [ ] 핵심 여정에 불필요한 단계 없는가?
- [ ] 기본값이 정말 '좋은 선택'인가?
- [ ] 복잡한 과정을 더 쪼갤 수 있는가?
- [ ] 처음부터 너무 많은 정보를 보여주진 않았는가?

**넛지 기회:**
- [ ] 결정적 순간에 사회적 증거를 활용하고 있는가?
- [ ] 지금 행동해야 할 이유가 분명한가?
- [ ] 호기심을 자극하는 장치가 있는가?

## S — Store: 긍정적 기억을 습관으로 만들어라

> 긍정적 Store가 반복되면 시스템 2(의식적 사고) → 시스템 1(무의식적 습관)으로 전환

### 긍정적 기억의 4가지 요소

| 요소 | 설명 | 실전 적용 |
|------|------|-----------|
| **Clear Feedback** | 행동 결과를 즉시, 명확히 알려줌 | 성공 애니메이션, 진행 상태 표시, 완료 확인 |
| **Reassurance** | 올바른 행동이라는 확신 → 심리적 안정 | 결제 전 요약, 보안 배지, "언제든 취소 가능" |
| **Feeling of Caring** | 제품이 나를 아끼고 있다는 느낌 | 맞춤 추천 이유 설명, 과사용 경고, 따뜻한 톤 |
| **Delighters** | 기대 이상의 감동 | 이탈 예상 시점에 관대한 제안, 마일스톤 축하 |

### Store 체크리스트

- [ ] 사용자의 모든 핵심 행동에 즉각적 피드백이 있는가?
- [ ] 불안할 수 있는 순간에 안심 요소가 있는가?
- [ ] 제품이 사용자를 케어한다는 느낌을 주는가?
- [ ] 예상치 못한 감동 요소가 있는가?
