# 제품 심리학 통합 리뷰 체크리스트

PM이 바이브코딩으로 만든 화면, 플로우, 카피를 리뷰할 때 사용하는 체크리스트.

---

## 🔍 Quick Review (5분 리뷰)

아래 5가지만 확인해도 주요 문제를 잡을 수 있다:

1. **한 화면에 선택지가 6개를 넘지 않는가?** (Block - Hick's Law)
2. **혜택이 기능 설명보다 먼저 보이는가?** (Interpret - Clear Benefit)
3. **다음에 해야 할 행동이 명확한가?** (Act - Remove Options)
4. **행동 후 피드백이 즉시 나오는가?** (Store - Clear Feedback)
5. **이 화면을 사용자가 보고 있다면 후회할 것 같은 부분이 있는가?** (Ethics - Regret Test)

---

## 📋 Full Review (상세 리뷰)

### Phase 1: Block — 뇌의 필터를 통과하는가?

**차단 요소 제거:**
- [ ] 1초 이상 "이게 뭐지?" 고민하게 만드는 요소 없음
- [ ] 시스템 2 없이 직관적 이해 가능
- [ ] 선택지 6개 이하
- [ ] 광고처럼 보이는 영역에 중요 정보 없음
- [ ] 배너 상단/우측 사이드바에만 핵심 정보가 있지 않음

**주의 유도 요소:**
- [ ] 프라이밍: 메인 CTA 전에 관련 맥락 제공
- [ ] 개인화: "나와 관련된" 느낌을 주는 장치
- [ ] 패턴 브레이크: 기존 반복을 깨는 요소

### Phase 2: Interpret — 가치가 명확히 전달되는가?

- [ ] 익숙한 UI 패턴 사용 (Familiarity)
- [ ] 인지 부하 최소화 (Cognitive Load)
- [ ] 혜택 먼저 제시 (Clear Benefit)
- [ ] 비교 기준점 제공 (Anchoring)
- [ ] 손실 시각화 (Loss Aversion)
- [ ] 핵심 기능 발견 용이 (Discoverability)
- [ ] 대기 시간의 가치 부여 (Labor Illusion)

### Phase 3: Act — 행동 장벽이 없는가?

**마찰 점검:**
- [ ] 불필요한 단계 없음 (Remove Options)
- [ ] 기본값이 좋은 선택 (Valid Defaults)
- [ ] 복잡한 과정 쪼개짐 (Split Steps)
- [ ] 정보의 점진적 공개 (Reveal Features)

**넛지 확인:**
- [ ] 사회적 증거 활용 (Social Proof)
- [ ] 호기심 자극 장치 (Curiosity Gap)
- [ ] 지금 행동해야 할 이유 (Scarcity)

### Phase 4: Store — 긍정적 기억이 남는가?

- [ ] 핵심 행동에 즉각적 피드백 (Clear Feedback)
- [ ] 불안 순간에 안심 요소 (Reassurance)
- [ ] 케어 받는 느낌 (Feeling of Caring)
- [ ] 기대 이상의 감동 (Delighters)

### Phase 5: BMAP — 행동 조건이 충족되는가?

- [ ] 동기(M): 감정적 동기가 충분한가? (기대/감각/소속)
- [ ] 능력(A): 시간/비용/노력/인지/숙련 장벽이 없는가?
- [ ] 자극(P): 적절한 타이밍에 적절한 시그널이 있는가?

### Phase 6: Peak-End — 여정의 정점이 설계되었는가?

- [ ] 가장 강렬한 긍정 순간(Peak)이 존재하는가?
- [ ] 부정적 저점(Pit)이 메워졌는가?
- [ ] 경험 전환 지점(Transition)이 명확한가?
- [ ] 핵심 단계의 순서가 최적인가? (리워드를 앞당길 수 있는가?)
- [ ] 앞단(트래픽 70-80%)에서의 Peak가 충분히 높은가?

### Phase 7: 윤리 — 사용자를 존중하는가?

- [ ] **Regret Test**: 사용자가 이 화면을 보고 있다면 후회할 부분 없는가?
- [ ] **Black Mirror Test**: 극단적으로 악용될 수 있는 설계가 없는가?
- [ ] **In Real-Life Test**: 이 제품이 실제 사람이라면 친절한가, 무례한가?
- [ ] 시간을 절약해주는가? (낭비 아닌지)
- [ ] 주의를 존중하는가? (방해 아닌지)
- [ ] 인간적 가치를 반영하는가? (주주 이익만이 아닌지)

## 🎯 상황별 중점 체크 가이드

| 작업 | 중점 Phase |
|------|-----------|
| 랜딩페이지 | Block + Interpret + Act |
| 온보딩 | Act (Split Steps, Reveal Features) + Store |
| 결제 플로우 | Act (Reduce Friction) + Store (Reassurance) |
| 가격 페이지 | Interpret (Anchoring, Loss Aversion) |
| 푸시 알림 | Block (Personalization) + Act (Scarcity) |
| 에러/빈 화면 | Store (Clear Feedback, Caring) |
| 기능 우선순위 | Peak-End (Elevate Peak 우선) |
| A/B 테스트 가설 | BMAP으로 변수 분리 |
