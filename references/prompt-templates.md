# PM 바이브코딩용 프롬프트 템플릿

PM이 바이브코딩 중 바로 복사해서 사용할 수 있는 프롬프트 모음.

---

## 1. 6P 스토리보드 생성

```
UX 심리학 전문가로서 우리 서비스 [서비스명]의 고객 여정을 6P 스토리보드로 만들어줘.

## 서비스 설명
[서비스 한 줄 설명]

## 타겟 사용자
[페르소나 간단 설명]

## 작성 원칙
- 주인공은 고객 (제품이 아님)
- 표정과 상황 위주의 시각적 언어 사용
- 불편함과 해결 후 안도감 대비
- 해피엔딩부터 역순으로 작성

## 질문 리스트 (순서대로 답변)
1. Happy Ending — 고객이 제품을 써서 도달한 최고의 순간은?
2. Problem — 앱을 켜기 전 현실에서 처음 느낀 불편/고통은?
3. Emotion — 그 순간 간절히 외치던 '나아지고픈 마음'은?
4. Action — 고객이 제품과 처음 만난 접점은?
5. Struggle — 해결 과정에서 가장 짜증나게 만든 장애물은?
6. Attempt — 어떤 '결정적 기능'이 장애물을 부수고 확신을 줬나?

## 출력
위 질문 답변 후, 6컷 스토리보드 (각 칸에 말풍선 1개 + 표정 변화)
순서: Problem → Emotion → Action → Struggle → Attempt → Happy Ending
```

## 2. B.I.A.S 기반 UI 리뷰

```
제품 심리학 전문가로서 아래 화면/플로우를 B.I.A.S 프레임워크로 분석해줘.

## 분석 대상
[화면 설명 또는 스크린샷]

## B.I.A.S 분석 기준
각 단계별로 점수(1-5)와 개선안을 제시해줘:

### Block (뇌의 필터 통과)
- 선택지 개수, 인지 복잡도, 광고 유사성, 개인화 여부

### Interpret (가치 해석)
- 혜택 가시성, 인지 부하, 패턴 친숙도, 기준점 제공

### Act (행동 유도)
- 마찰 요소, 기본값 설계, 단계 복잡도, 넛지 활용

### Store (기억 저장)
- 피드백 명확성, 안심 요소, 케어 느낌, 감동 요소

## 출력 형식
각 단계별: 현재 점수 → 문제점 → 구체적 개선안 (코드 포함)
```

## 3. BMAP 기반 행동 분석

```
BMAP 모델로 사용자가 [특정 행동]을 하지 않는 이유를 분석해줘.

## 원하는 행동
[예: 회원가입 완료, 첫 구매, 친구 초대 등]

## 현재 상황
[현재 전환율, 이탈 지점 등]

## BMAP 분석
### Motivation (동기) — 감정 기반
- 기대(Anticipation): 사용자가 기대하는 것은?
- 감각(Sensation): 즐거운 경험이 있는가?
- 소속(Belonging): 소속감/인정 욕구를 자극하는가?

### Ability (능력) — 5가지 장벽
- 시간: 얼마나 걸리는가?
- 비용: 돈이 드는가?
- 신체적 노력: 클릭/스크롤이 많은가?
- 인지적 노력: 복잡한가?
- 숙련도: 익숙한 행동인가?

### Prompt (자극) — 타이밍과 맥락
- 명시적 자극: 알림, 버튼, 배너가 적절한가?
- 암묵적 자극: 기억/연상으로 떠오르는 장치가 있는가?

## 출력
가장 약한 요소 식별 → 개선 우선순위 → 구체적 실행안
```

## 4. Peak-End 저니맵 재설계

```
Peak-End Rule을 적용하여 [서비스명]의 사용자 여정을 재설계해줘.

## 현재 여정
[주요 단계 나열]

## 분석 요청
1. 현재 여정에서 Peak(고점)과 Pit(저점)을 식별
2. 각 Transition(전환 지점) 표시
3. 아래 4가지 전략 적용한 개선안 제시:
   - Elevate the Peak: 고점을 더 높이기
   - Repair the Worst: 저점을 메우기
   - Improve Transitions: 전환 지점 명확히 하기
   - Reorder Steps: 리워드를 앞당기는 등 순서 재정렬

## 주의사항
- 트래픽의 70-80%가 앞단에 모여있으므로 앞단 Peak 우선
- 이탈하는 사람이 아닌 core value를 느끼는 고객에 집중
```

## 5. 전환율 개선 가설 수립

```
[서비스명]의 [특정 퍼널]에서 전환율을 개선하기 위한 가설을 수립해줘.

## 현재 데이터
[퍼널별 전환율, 이탈 지점 등]

## 분석 프레임워크
다음 순서로 분석해줘:

1. **질문 전환**: "이탈 지점이 어디?" → "마음이 왜 바뀌었지?"
   - "왜 이탈했을까?" → "왜 클릭했을까?"
   
2. **6P 스토리보드**: 이탈 사용자의 맥락과 감정 추정

3. **BMAP 분석**: 행동하지 않은 이유 (M/A/P 중 뭐가 부족?)

4. **B.I.A.S 체크**: 각 단계에서 놓친 것

5. **Peak-End 관점**: 여정의 어디에서 Peak를 만들 수 있나?

## 출력 형식
가설: [변수]를 [변경]하면, [메트릭]이 [방향]할 것이다.
근거: [심리학 원칙]에 의하면...
실행안: [구체적 변경 사항]
```

## 6. 윤리적 디자인 검토

```
아래 기능/화면에 대해 윤리적 디자인 검토를 해줘.

## 검토 대상
[기능/화면 설명]

## 3가지 테스트
1. **Regret Test**: 사용자가 이 설계를 알게 되면 후회할 부분이 있는가?
2. **Black Mirror Test**: 극단적으로 악용될 수 있는 부분은?
3. **In Real-Life Test**: 이 제품이 사람이라면 어떤 사람인가?

## 추가 체크
- Humane Design Principles 준수 여부
- Second-order Effect (의도치 않은 부작용) 가능성
- 다크 패턴 존재 여부

## 출력
위험 수준(🟢 안전 / 🟡 주의 / 🔴 위험) + 개선 권고
```

## 7. 온보딩 플로우 설계

```
[서비스명]의 온보딩 플로우를 제품 심리학 원칙을 적용해서 설계해줘.

## 서비스 설명
[서비스 한 줄 설명]

## 적용 원칙
- **Act**: Split Steps (쉬운 선택 먼저) + Reveal Features (점진적 공개)
- **Block**: 1 thing per 1 page, 선택지 최소화
- **Interpret**: Clear Benefit (각 단계의 혜택 먼저)
- **Store**: Clear Feedback (각 단계 완료 피드백)
- **BMAP**: 첫 행동의 Ability 장벽을 최소화

## 출력
단계별 화면 구성 + 각 화면에 적용된 심리학 원칙 주석
```
